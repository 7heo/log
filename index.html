    C# Properties

    2013/11/18 - 0315 GMT+1

    So, I've done some C# recently. And, since I'm not really used to the language, I read the MSDN documentation a lot, and also stackoverflow, when I don't find the info there.

    But rarely, I don't find the info on either of these websites. It might be because I don't ask the right question, or because it's about a trivial problem that most .NET developers overcome in seconds.

    Here is one problem I had some hard time to overcome (followed, of course, by the solution):

    Given objects you can access via the "Square bracket" operators (i.e. '[]') on another object, how to list all the (children) objects you can access?

    First of all, I've tried to make my code throw an exception, so I could check the existing variables and their values; hopefully finding the correct data structure in the list. This was a complete failure. The indexed values are not listed under an object.

    Then I've tried all sort of black magic tricks I've found all around the web, without success. I've noticed that my object had several properties, including "Container". So I've tried to access that container, to list all of its elements, and so on. Again, that was a complete failure. I'm not really used to the .NET framework, but it seems to make very little sense to search for properties in components.

    And then I have found the syntax for foreach (good old foreach you also use in PHP to access arrays). I've tried to access my "indexed" object via foreach, but again, this was a failure. Though, this time, I had a bit more information: Error: foreach statement cannot operate on variables of type '<object>' because '<object>' does not contain a public definition for 'GetEnumerator'. NOW I had some information about what to search. An object that has a GetEnumerator method.

    Given that new information, I have searched all the main object properties for one that had a GetEnumerator method. I've found it to be called "Properties". Meh. Obvious name should have been obvious.

    So the key to this problem was to understand the implicit link between the Properties property and the Indexer access in that case.

    I guess it should have been really straightforward for an experienced .NET programmer. Still, for me, that's the kind of things that would require a more extensive documentation, or even better, a corresponding data visualization in the IDE.

    Microsoft won't ever change

    2013/11/09 - 0300 GMT+1

    Recently, I've witnessed the (pathetic?) efforts that Microsoft does to improve their image, to innovate, and so on. As a result, when I want to quickly prototype application idea, I sometimes use C#.

What happened is that I created a new project, coded a bit, and when I was satisfied with my work, after one hour or so; I did leave the IDE open in order to surf a bit, and quickly got back at it. Needless to say, I'm used to save my work, and I pressed [Ctrl] + [s] every five minute or so; while I was coding.

As I'm not a big fan of the compulsory identification practices that spread like the pest, I clicked on cancel, exactly the same way I did when I opened the IDE, hoping to return to my work.
